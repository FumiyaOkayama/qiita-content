# GitHub Copilotとqiita cliを使って爆速で記事を作成/公開する

GitHub Copilotとqiita cliを使って、記事の作成から公開までを効率化する方法を紹介します。

## 前提条件

- GitHub CopilotがVScode上で有効になっていること
- qiita cliがインストールされていること

## VScodeの設定

1. VScodeでGitHub Copilotを有効にします。
2. VScodeから [左下の歯車] > [Setttings] を押下し、 「Copilot」と検索し、markdownを[true]に設定します。
  ![image-1.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/4187508/ac9c9004-61b4-41aa-bbcf-4ecdbf440791.png)
3. マークダウンのファイルを開きカーソルを置くと、GitHub Copilotが提案を開始します。[Tab]キーで提案を受け入れます。
  ![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/4187508/8af158db-fc38-4d9a-85e2-f596627c0923.png)

## Qiita CLIの設定

https://github.com/increments/qiita-cli を参照してください。
masterブランチかmainブランチにpushすると、自動的にQiitaへ投稿されます。

:::note note
個人的に行った設定です。
- qiita.config.jsonの設定で、`"includePrivate": true`に変更して、非公開記事も同期の対象にしました。
- draftブランチを作成し、執筆中の記事を管理するようにしました。
:::
