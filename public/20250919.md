# GitHub Copilotとqiita cliを使って爆速で記事を作成/公開する

GitHub Copilotとqiita cliを使って、記事の作成から公開までを効率化する方法を紹介します。

## 前提条件

- GitHub CopilotがVScode上で有効になっていること
- qiita cliがインストールされていること

## VScodeの設定

1. VScodeでGitHub Copilotを有効にします。
2. VScodeから [左下の歯車] > [Setttings] を押下し、 「Copilot」と検索し、markdownを[true]に設定します。
  ![image-1.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/4187508/ac9c9004-61b4-41aa-bbcf-4ecdbf440791.png)
3. マークダウンのファイルを開きカーソルを置くと、GitHub Copilotが提案を開始します。[Tab]キーで提案を受け入れます。
  ![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/4187508/8af158db-fc38-4d9a-85e2-f596627c0923.png)

## Qiita CLIの設定

https://github.com/increments/qiita-cli を参照してください。
masterブランチかmainブランチにpushすると、自動的にQiitaへ投稿されます。

:::note note
個人的に行った設定です。
- qiita.config.jsonの設定で、`"includePrivate": true`に変更して、非公開記事も同期の対象にしました。
:::

## 記事作成~投稿・同期してみた

1. 新規記事の作成
  ```bash
  npx qiita new 記事のファイルのベース名
  ```
  qiita-contentディレクトリに、記事のテンプレートが作成されます。

2. VScodeで記事の編集
  GitHub Copilotの提案を受け入れながら、記事を執筆します。

3. プレビュー画面の表示
  ```bash
  npx qiita preview
  ```
  http:localhost:8888 にアクセスし、左メニューから記事を選択すると、プレビュー画面が表示されます。

4. Qiitaへの投稿
  masterブランチかmainブランチにpushすると、自動的にQiitaへ投稿されます。
  投稿されると、github actionsが動き、Qiitaへ投稿されます。投稿後、updated_at や id が自動的に設定されます。
  ![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/4187508/24aba5d0-11e4-4127-adc0-e176bb09e4a6.png)

5. 投稿記事の同期
  ```bash
  npx qiita pull
  ```
  Qiitaに投稿されている記事をローカルに同期します。
  今回の場合、以下のような構成になりました。
  ```tree
  public
  ├── .remote
  |    └── 7478761dcc7515a551a2.md # idが設定され同期された記事: .remote配下なので差分管理の対象外
  └── 20250919.md # npx qiita new で作成した記事
  ```

## まとめ
GitHub Copilotとqiita cliを組み合わせることで、記事の作成から公開までのプロセスを大幅に効率化できます。ぜひ試してみてください。
